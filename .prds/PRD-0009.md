# Summary

Make Conclave deployable to a public host (Fly.io the immediate target) and reachable by real
clients over TLS. The gating item is **client-side TLS**: the shipped bridge/CLI cannot currently
dial `wss://`, so the design's TLS-at-the-edge model is unreachable. On top of that: a container
image, a `fly.toml` with a persistent volume for the store, a guard against the silent in-memory
footgun, production-grade logging, and a documented clone-to-running deploy with a key-bound admin.

# Problem

The stated next step — the operator deploying their own server and driving it from local sessions —
is blocked by a single confirmed gap: `tokio-tungstenite` is built with **no TLS feature**, so
`connect_async("wss://…")` fails and the client can only reach `ws://` (`client.rs:136`,
`control.rs:85`). The design assumes TLS terminates at a cloudflared/fly.io edge with the origin
speaking plain WS, but nothing lets a client actually traverse that edge. Deployment also has no
container image, no platform manifest, no volume story for the SurrealKV data dir (and a footgun
where a missing `--data-dir` silently runs in-memory and wipes state on restart), and
non-production logging.

# Technical Approach

```
bridge / CLI  ──wss:// (rustls, T-001)──►  Fly.io TLS edge  ──ws:// (internal_port)──►  conclave serve
                                                                                          │
                                                                              Fly volume ─┘ (SurrealKV --data-dir, T-002/T-004)
```

Enable `tokio-tungstenite`'s `rustls-tls-webpki-roots` feature and let `connect_async` select TLS by
URL scheme; keep the server plain. Build the image with cargo-chef for cached deps and run
`conclave serve` as non-root, pointing `--data-dir` at a mounted Fly volume declared in
`fly.toml [mounts]`. Make `serve` refuse to start without a persistent data dir unless an explicit
`--ephemeral` flag is given. Wire `RUST_LOG`/`EnvFilter` and TTY-aware coloring. Document the deploy
end-to-end with the admin bound to the operator's key (PRD-0007 T-002).

# Assumptions

- PRD-0007 (server hardening) is complete before a *public* deploy; a private single-operator deploy may proceed earlier at the operator's risk.
- Fly.io's proxy provides TLS termination and a TCP health check suffices for the WS-only origin (no HTTP route by design).
- Graceful SIGTERM shutdown already works (verified in review: serve exits ~0.06s after signal).

# Constraints

- The server keeps its plain HTTP/WS origin (no rustls in `serve`) — TLS is edge-only (DESIGN §11/§12).
- v1 is a single `serve` instance; HA/horizontal scale stays deferred (DESIGN §19).
- Persistence remains embedded SurrealDB over SurrealKV; no external DB dependency.

- `src/server/wss.rs` (:51) and `src/bin.rs` (serve args, :29 logging) — store mode + observability.

# Non-Goals (MVP)

- Multi-instance / HA, cross-instance presence, and backups beyond the single mounted volume (DESIGN §19).
- Server-side rustls (TLS stays at the edge); E2E payload encryption (v2).
- Bridge/CLI reliability fixes (PRD-0008), though they materially improve the deploy-testing experience.
