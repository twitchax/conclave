# Summary

Turn the PRD-0015 handle-collision diagnosis from advice into action. Today a diagnosed
supersede fight prints "start one with a distinct `--as`" and goes quiet — but bridges are
launched from user-scoped shared MCP config, so same-directory sessions (a fleet: the live
`dotagent1..6` storm that motivated this) can never follow that advice. A bridge whose handle
was *defaulted* from the directory name now renames itself (`dotagent-2`, `-3`, …) when the
breaker trips, reconnects under the new name, and tells its session. Explicit `--as` handles
keep the advisory behavior: the bridge never overrides a name the user chose.

# Non-Goals

- Server-side handle allocation (the server referees supersede correctly; naming is the
  client's job and keeps working against old servers — no wire change).
- Renaming explicit `--as` handles, ever.
- Handle stability across bridge restarts (a fresh bridge re-derives the default and, if it
  collides, re-disambiguates — same as today's supersede semantics).
